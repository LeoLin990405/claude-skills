# Business Proposal Template

A structured template for creating professional business proposals. Use with the [docx skill](../skills/docx/SKILL.md) to generate formatted Word documents.

---

## Proposal Structure

### 1. Cover Page

```
[Company Name / Logo]
[Proposal Title]

Prepared for: [Client / Recipient Name]
Prepared by: [Author / Company Name]
Date: [Date]
Proposal #: [Reference Number]
Valid until: [Expiration Date]
```

### 2. Executive Summary

One page maximum. Write this LAST.

```
THE OPPORTUNITY
- What the client needs (in their language)
- Why it matters now

OUR APPROACH
- High-level solution summary (2-3 sentences)
- Key differentiator

EXPECTED OUTCOMES
- Outcome 1: [Quantified benefit]
- Outcome 2: [Quantified benefit]

INVESTMENT
- Total cost: [Amount or range]
- Timeline: [Duration]
- ROI: [Expected return or payback period]
```

### 3. Problem Statement

```
CURRENT SITUATION
- Description of the client's current state
- Pain points and challenges
- Impact of inaction (cost of doing nothing)

EVIDENCE
- Data points that quantify the problem
- Market trends or competitive pressures
- Client-provided information or research

ROOT CAUSES
- Why the problem exists
- What has been tried before
- Why previous approaches fell short
```

### 4. Proposed Solution

```
OVERVIEW
- Solution description in plain language
- How it addresses each pain point identified above

APPROACH
- Methodology / framework
- Key activities and deliverables
- Technology or tools involved

DIFFERENTIATORS
- What makes this approach unique
- Relevant experience or case studies
- Why this solution over alternatives
```

### 5. Scope of Work

```
IN SCOPE
- Deliverable 1: [Description + acceptance criteria]
- Deliverable 2: [Description + acceptance criteria]
- Deliverable 3: [Description + acceptance criteria]

OUT OF SCOPE
- [Item explicitly excluded]
- [Item explicitly excluded]

ASSUMPTIONS
- [Assumption about client resources]
- [Assumption about access or data]
- [Assumption about timeline]
```

### 6. Timeline

```
PHASE 1: [Name] — Week 1-2
- Milestone: [Deliverable]
- Key activities: [List]

PHASE 2: [Name] — Week 3-6
- Milestone: [Deliverable]
- Key activities: [List]

PHASE 3: [Name] — Week 7-8
- Milestone: [Deliverable]
- Key activities: [List]

KEY DATES
- Kickoff: [Date]
- Mid-point review: [Date]
- Final delivery: [Date]
```

### 7. Pricing

```
OPTION A: [Package Name]
- Scope: [What's included]
- Investment: [Amount]
- Payment terms: [Schedule]

OPTION B: [Package Name] (Recommended)
- Scope: [What's included]
- Investment: [Amount]
- Payment terms: [Schedule]

OPTION C: [Package Name]
- Scope: [What's included]
- Investment: [Amount]
- Payment terms: [Schedule]

NOTES
- What's included in all options (e.g., revisions, support)
- Additional costs if applicable (travel, licenses)
- Payment milestones tied to deliverables
```

### 8. Team

```
PROJECT LEAD: [Name]
- Role and responsibilities
- Relevant experience

TEAM MEMBER: [Name]
- Role and responsibilities
- Relevant experience

CLIENT RESPONSIBILITIES
- [What the client needs to provide]
- [Availability requirements]
- [Decision-making authority]
```

### 9. Why Us

```
RELEVANT EXPERIENCE
- Case study 1: [Client, challenge, result]
- Case study 2: [Client, challenge, result]

TESTIMONIALS
- "[Quote]" — [Name, Title, Company]

CREDENTIALS
- [Certifications, awards, partnerships]
```

### 10. Terms and Conditions

```
- Validity period
- Intellectual property
- Confidentiality
- Termination clause
- Liability limitations
```

### 11. Next Steps

```
TO PROCEED
1. [Sign and return proposal by Date]
2. [Schedule kickoff meeting]
3. [Provide access to X]

CONTACT
- [Name]: [Email] | [Phone]
```

---

## Formatting Guidelines

| Element | Style |
|---------|-------|
| Title | Heading 1, bold, 18pt |
| Section headers | Heading 2, bold, 14pt |
| Body text | Normal, 11pt, 1.15 line spacing |
| Pricing tables | Bordered, header shaded, amounts right-aligned |
| Page numbers | Bottom right |
| Brand colors | Use client's colors for headers if known |

## Usage

```
"Write a proposal for the website redesign project using the proposal template"
→ Claude loads docx skill + this template, generates structured .docx file
```
